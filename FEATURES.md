# FlashOfThought (语音想法助手) 功能与架构文档

## 1. 项目简介
**FlashOfThought** 是一款由大语言模型（LLM）驱动的智能语音笔记与灵感管理助手。它能够通过语音或文字快速捕捉用户的“闪念”，利用 AI 自动提取核心观点、构建结构化笔记，并提供知识图谱可视化、语义检索、AI 深度分析以及交互式问答等功能，帮助用户将零散的灵感转化为有价值的知识库。

---

## 2. 核心功能模块

### 🎙️ 1. 灵感录入 (Record Idea)
支持多模态的信息捕捉，并由 AI 自动对内容进行结构化整理：
*   **多源输入支持**：
    *   **📤 上传文件**：支持上传 `.mp3`, `.wav`, `.m4a`, `.aac` 等格式的音频文件。
    *   **🎤 麦克风录音**：网页端直接录制语音。
    *   **📝 文字输入**：直接输入长文本想法。
    *   **⚡ 闪念指令模式**：通过语音指令（如：“闪念，记录一个新的AI想法…”）快速唤醒并记录。
*   **AI 自动结构化**：录入的原始文本或语音转写结果将被 AI 自动整理为结构化卡片，包含：**标题**、**标签**、**核心摘要**、**核心观点**、**关键功能** 和 **可能应用**。
*   **第二大脑关联**：在录入新想法时，系统会通过语义搜索（RAG）自动在侧边栏推荐库中已有的相似灵感。

### 🧠 2. 知识回顾与管理 (Knowledge Review)
沉淀灵感并提供多种回顾方式：
*   **🔎 智能搜索**：基于 ChromaDB 的向量语义检索，支持通过自然语言提问（如“我上次关于AI的计划是什么？”）精准找回笔记。
*   **💬 聊天回顾**：与知识库进行对话（RAG 机制），AI 会基于你过往的所有笔记作为上下文来回答你的问题。
*   **📊 AI 周报**：自动汇总过去 7/14/30 天的笔记，生成包含“核心关注点”、“主题分布”和“重点想法”的 AI 摘要报告。
*   **📚 笔记管理**：浏览近期笔记，支持直接编辑（标题、摘要、标签）和删除功能。

### 🤖 3. AI 深度分析 (Deep Analysis)
针对单条笔记，提供深度的商业与执行面扩展：
*   **✨ 扩展想法**：AI 自动为当前灵感补充具体功能、产品形态、商业模式及技术实现建议。
*   **📅 生成路线**：根据灵感内容，自动规划出分阶段的落地执行 Roadmap（路线图）。
*   **📊 灵感评分**：从 **创新性**、**商业价值**、**技术难度**、**可行性** 四个维度对灵感进行打分（满分 5 分），并给出详细的评估理由。

### 🌌 4. 知识图谱 (Knowledge Graph)
*   **可视化星系**：使用 ECharts 将知识库渲染为 3D/2D 节点图谱。
*   **层级结构**：呈现为 `核心 Idea (中心)` ➡️ `功能标签 (聚类)` ➡️ `知识卡片 (叶子节点)` 的拓扑结构。
*   **交互控制**：支持“力导向图”与“环形布局”切换，支持节点拖拽、悬停预览摘要、点击高亮关联。

## 3. 技术架构设计

### 后端架构 (Backend)
*   **Web 框架**：基于 `FastAPI` 构建的高性能 RESTful API。
*   **核心服务 (Services)**：
    *   `llm_service`：负责结构化提取、内容扩展、打分、对话以及周报生成。
    *   `oss_service`：对象存储服务，处理音频文件的云端上传并生成 URL。
    *   `rag_service`：基于 **ChromaDB** 的向量检索与本地数据存储服务。
*   **语音转写 (ASR) 容灾策略**：上传链路中应用了模型回退机制，默认优先调用更快的 `qwen3-asr-flash`，若遇 OSS 签名 URL 导致 `InvalidParameter` 错误，系统会自动无缝回退至 `paraformer-v2` 模型以保障转写成功率。

### 前端架构 (Frontend)
*   **生产 UI (Python)**：基于 `Streamlit` 构建，使用自定义 CSS 实现了暗黑科技风，集成了 `streamlit-mic-recorder` (语音录制) 与 `streamlit-echarts` (图谱渲染)。
