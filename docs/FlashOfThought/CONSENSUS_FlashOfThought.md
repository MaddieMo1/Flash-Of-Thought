# CONSENSUS_FlashOfThought: 语音笔记智能助手共识文档

## 1. 核心共识 (Core Consensus)

**项目目标**: 构建一个名为 **FlashOfThought** 的智能语音笔记应用，能够将用户的语音想法快速转化为结构化的知识，并支持向量化存储、检索以及通过 MCP 协议供外部调用。

**核心功能**:
1.  **语音转结构化笔记**: 上传录音 -> 转文字 (ASR) -> 结构化整理 (LLM) -> 生成 Markdown 笔记。
2.  **知识库构建**: 将整理后的笔记向量化，存入本地向量数据库。
3.  **多端调用**:
    *   **Web UI**: 提供直观的界面供用户录音、查看和检索。
    *   **API**: 提供 RESTful API 供第三方集成。
    *   **MCP Server**: 作为 MCP 工具集，供 AI Agent (如 Trae, Claude Desktop) 直接调用。

## 2. 技术栈选型 (Tech Stack)

| 组件 | 选型 | 理由 |
| :--- | :--- | :--- |
| **语言** | Python 3.10+ | AI 生态最成熟，MCP SDK 支持完善 |
| **Web UI** | **Streamlit** | 快速构建数据驱动的交互界面，适合 AI 原型 |
| **后端 API** | **FastAPI** | 高性能，易于生成 OpenAPI 文档，原生支持异步 |
| **向量数据库** | **ChromaDB** | 轻量级本地嵌入式数据库，部署简单，无额外运维成本 |
| **对象存储** | **阿里云 OSS** | 稳定存储原始音频文件，通过 `.env` 配置 |
| **AI 服务** | **阿里云 DashScope (Qwen)** | 通义千问 Qwen-Plus (文本), Qwen-Audio (ASR), Text-Embedding-v4 (向量) |
| **协议** | **MCP (Model Context Protocol)** | 标准化 AI 工具调用协议 |

## 3. 验收标准 (Acceptance Criteria)

1.  **功能性**:
    *   用户可以通过 Web 界面上传音频，并在 30 秒内获得结构化的笔记。
    *   笔记内容准确反映语音意图，包含：摘要、详细内容、行动建议。
    *   用户可以通过自然语言检索过去的笔记（基于语义）。
    *   MCP 客户端（如 Trae）可以调用 `add_idea` 和 `query_ideas` 工具。
2.  **稳定性**:
    *   API 错误处理完善，Key 缺失或网络异常时有清晰提示。
    *   大文件（>10MB）上传处理机制（或前端限制）。
3.  **代码质量**:
    *   Python 代码符合 PEP8 规范。
    *   核心逻辑有单元测试覆盖。
    *   敏感信息（API Key）严格通过 `.env` 管理。

## 4. 边界与限制 (Constraints)

*   **本地运行**: 数据库和应用默认在本地运行，不涉及复杂的云端部署架构。
*   **单用户模式**: 初始版本暂不设计多用户鉴权体系，假定为个人使用。
*   **并发限制**: 依赖 DashScope API 的并发配额。
